# Aetra Trademark Policy

This policy applies to the Aetra name, wordmark, logo, and other identifying
marks associated with the project.

## Allowed Uses

- Referencing the project in factual discussions, comparisons, benchmarks, or
  compatibility notes.
- Describing an unmodified upstream fork as a fork of Aetra.
- Using the project name in a nominative way, such as "compatible with Aetra"
  or "built for Aetra".

## Restricted Uses

- Presenting a fork, rebrand, or downstream distribution as the official
  Aetra project.
- Using the Aetra name or logo in a way that implies endorsement, sponsorship,
  or official status without permission.
- Replacing the upstream project identity with a near-identical name, logo, or
  release branding intended to confuse users.

## Release and Branding Requirements

- Forks should clearly identify themselves as forks or derivatives.
- Official releases should preserve the Aetra logo and project notices.
- Modified distributions should state that they are not official Aetra releases.
- Any external use of the logo outside ordinary project references requires
  permission from the project owner.

## Notes

- This policy does not replace the Apache License 2.0.
- Copyright covers the code.
- Trademark policy covers the name, logo, and source-identifying branding.
